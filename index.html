Sales Data Trend Analysis and Reporting

This project involved creating a comprehensive sales report to help the Sales Director analyze trends, uncover patterns, and assess key business metrics. Raw sales data was processed, normalized into a star schema, and visualized using Power BI to provide dynamic insights.
Key Steps:
Data Ingestion & Normalization:
Sales data was imported into Google BigQuery and structured using SQL into a star schema, optimizing for efficient querying.
Data Modeling:
Fact and dimension tables were linked in Power BI, with additional calculated measures for KPIs like Total Sales and Profit Margins.
Interactive Dashboard:
A Power BI dashboard was built to visualize sales trends, product performance, and customer insights, with filters for dynamic exploration. This project enables the Sales Director to interactively explore data and make data-driven decisions on sales performance and growth

Food Service Analytics Dashboard

This project focuses on creating a comprehensive analytics dashboard for a pizzeria, covering customer orders, inventory, and staff management. The workflow involves designing a custom SQL database, writing complex SQL queries, and integrating the database with a Business Intelligence (BI) tool to visualize key metrics. Key Steps: Database Design:
Built a normalized relational database to track orders, stock levels, and employee schedules using QuickDBD and PostgreSQL.
SQL Query Writing:
Developed queries to extract insights like total sales, inventory needs, and labor costs, leveraging SQL techniques such as data joins and aggregation.
BI Integration:
Connected the database to a BI tool (Tableau) to create interactive dashboards, enabling the business to monitor performance in real time. This end-to-end solution showcases essential skills in database design, SQL, and data visualization, providing actionable insights for the business.

Advanced Data Cleaning in PostgreSQL

This project focused on transforming a raw dataset into a clean, structured format using PostgreSQL, demonstrating essential data cleaning skills critical for data analysis. Key tasks included standardizing date formats, normalizing addresses, unifying categorical data, removing duplicates, and optimizing the dataset by dropping unnecessary columns.
Key Steps:
Standardized Date Formats:
Converted date and time to a uniform date-only format for consistent analysis.
Resolved Missing Addresses:
Filled missing property addresses by cross-referencing records with the same parcel_id.
Split Address Data:
Separated combined address fields into individual components (e.g., street, city, state) for better usability. Unified Categorical Data: Standardized inconsistent values in the sold_as_vacant column into 'Yes' or 'No' categories.
Removed Duplicates:
Identified and removed duplicate records using window functions, ensuring data accuracy. Optimized Dataset: Dropped unnecessary columns to streamline the dataset for analysis.
